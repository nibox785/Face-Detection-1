# Documentation Standards

## Purpose

This document defines the standards for creating and maintaining documentation.

## Writing Guidelines

- Use clear headings and short sections.
- Prefer plain language and consistent terminology.
- Separate purpose, scope, and responsibilities.
- Keep content focused on the topic at hand.
- Avoid implementation-specific details and unsupported assumptions.

## Quality Expectations

- State the intent of the document early.
- Use examples only when they improve understanding.
- Keep statements factual, specific, and easy to verify.
- Mark uncertain or temporary content clearly when necessary.

## Review Expectations

- Review documentation for clarity, consistency, and completeness.
- Ensure related documents remain aligned.
- Update documents when ownership, scope, or expectations change.
